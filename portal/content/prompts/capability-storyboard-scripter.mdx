  available), length, and audience below, script a capability

  A good storyboard script:
  - Hooks the audience in the first panel (the operational

  - Shows the capability in use, not the capability as a

  - Includes at least one failure-recovery panel — the moment

  - Builds tension before resolution — don't lead with the

Use this prompt to script a visual storyboard of a capability in use — the panels, beats, and narration that turn an operational scenario into something operators can react to before any platform is acquired. The script feeds into a visualisation tool (Luma, Nano Banana, or Forma) for the actual rendering — this prompt produces the what-to-show, not the imagery.

## Tips

- The "carry-over" field is what turns a panel set into a narrative. Without it you get 12 disconnected hero shots.
- The capability should be visible in use, not as a system diagram. If your storyboard could be replaced by a wiring diagram, the capability isn't being shown — it's being labelled.
- Send the panel-level prompts through to [Luma](/tools/luma-ai), [Nano Banana](/tools/nanobanana), or [Forma](/tools/autodesk-forma) for the actual visualisation. The script is the spec — those tools are the renderer.
